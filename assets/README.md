# Brand & README Assets

Drop these files here so the main README renders fully. **Visuals are the single biggest driver of stars** — a hero logo + a demo GIF above the fold convert browsers into stargazers.

## Required (referenced by README.md)

| File | What it is | Recommended spec |
|---|---|---|
| `logo.png` | The DM NATIVE mark shown at the top | ~512×512, transparent background, mint (`#00ffbb`) accent |
| `demo.gif` | A short loop of the app in action (open chat → send an encrypted message → reply) | 680px wide, < 10 MB, 5–10s loop |

## Strongly recommended

| File | What it is |
|---|---|
| `screenshot-chats.png` | Chat list |
| `screenshot-conversation.png` | A conversation with a few message types |
| `screenshot-anonymous.png` | An anonymous room |
| `banner.png` | Optional wide social banner (1280×640) for GitHub social preview + link cards |

## Tips

- Record `demo.gif` on a real device for authenticity; keep it tight (the first 3 seconds decide whether someone keeps watching).
- Use a consistent dark theme + mint accent so the visuals match the brand.
- Set the GitHub repo's **Social Preview** image (Settings → Social preview) to `banner.png` so shared links look premium.
- Compress the GIF (e.g., gifski) — a heavy GIF hurts load time and the above-the-fold impression.
